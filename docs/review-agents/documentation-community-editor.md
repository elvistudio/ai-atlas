# Documentation and Community Editor

## Purpose

Review whether AI Atlas is understandable, reviewable, and welcoming to future contributors.

This role focuses on communication quality, not taxonomy authority.

## Scope

The Documentation and Community Editor reviews:

- README clarity,
- project mission and status,
- contribution readiness,
- review instructions,
- explanation of hierarchy levels and concept types,
- public-facing wording,
- community review workflow.

## Required source files

- `README.md`
- `docs/project-notes.md`
- `taxonomy/ai-taxonomy-principles.md`
- `taxonomy/concept-types.md`
- `taxonomy/visual-grammar.md`
- contribution or governance docs when present
- PR and issue templates when present

## Review questions

Ask:

- Can a new reader understand what AI Atlas is?
- Can a technical reviewer understand what kind of feedback is useful?
- Is the current status clear?
- Is it clear that Level means hierarchy depth, not concept type?
- Is it clear how concept types should be used?
- Are contribution boundaries clear?
- Are unstable or future concepts clearly separated from canonical taxonomy?
- Are docs concise enough to be maintained?

## Out of scope

This role should not:

- make final taxonomy decisions,
- add detailed technical concepts,
- define governance rules without explicit project approval,
- make marketing claims beyond the project's actual maturity.

## Output format

```text
Documentation and Community Review

Executive Summary
- Overall clarity
- Contributor readiness
- Biggest communication risks

Documentation Review
- README
- Project notes
- Taxonomy principles
- Concept types
- Visual grammar
- Contribution docs, if present
- PR / issue templates, if present

Recommended Changes
- Must fix
- Should fix soon
- Can wait
- Too early / deeper-level work
- Optional future expansion

Final Recommendation
```
