# Level 3 Draft Status and Roadmap

> Level 3 is draft/review-only and is not part of the public canonical taxonomy yet.
> The current public canonical AI Atlas taxonomy remains Level 0–2.

## What Level 3 is intended to add

Level 3 is intended to introduce more specific concepts under existing Level 2 areas. Developing it requires careful review of names, concept types, hierarchy placement, cross-branch relationships, and relation targets.

Level 3 concepts may be promoted publicly only after explicit review, validation, and approval. Draft work is not promoted merely because it has been documented or structured.

## Current public status

- AI Atlas v0.1 Level 0–2 is the public canonical taxonomy.
- Level 3 remains draft/review-only and is not canonical.
- No Level 3 public generated view should be treated as official.
- No private Level 3 draft should be interpreted as an accepted public node or stable public identifier.

## Private draft progress

Private work is being used to test possible Level 3 workflows without changing the public taxonomy. Current progress includes:

- draft and package work for the Large Language Models branch;
- schema review and migration planning for multi-node LLM work;
- a private two-node standalone extraction pilot for part of the first LLM group;
- private structured draft work related to Text Generation and Natural Language Generation;
- private cross-branch drafts for some LLM-related concepts;
- deferred candidates where concept type, placement, or policy remains uncertain.

This summary does not publish raw private JSON, private review documents, or proposed node contents. It does not mean that any Level 3 candidate has been accepted, promoted, or assigned a stable public ID.

## Blockers before public promotion

Public Level 3 promotion still requires:

- a public Level 3 review and promotion process;
- practical rules for cross-branch placement;
- consistent handling of structured packages for multi-node branches;
- capability-category policy review;
- a consistent relation target policy;
- validation and generated-view strategies suitable for public Level 3.

Until these blockers are resolved, Level 3 remains review-only and not promoted.

## How community reviewers can help

- Review the public Level 0–2 taxonomy first.
- Open GitHub issues for missing Level 2 coverage or unclear boundaries.
- Propose Level 3 candidates through issues rather than direct canonical additions.
- Prefer stable, academically recognizable, and technically useful concepts.
- Avoid products, companies, temporary marketing labels, and concrete model versions unless they are explicitly in scope.

For the current review boundaries, see the [public review guide](../../PUBLIC_REVIEW.md) and the [Level 3 draft policy](../../taxonomy/level-3-draft-policy.md).
