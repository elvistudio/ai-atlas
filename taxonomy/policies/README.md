# AI Atlas Taxonomy Policy Drafts

Status: draft policy index / under review / not canonical

The files in this directory are policy drafts for review. They do not change canonical taxonomy by themselves and do not authorize Level 3 promotion.

## Draft policies

- [Stable Taxonomy Node ID Convention](stable-id-convention.md) — proposes stable semantic IDs that do not encode mutable hierarchy placement or concept type. Tracks [issue #15](https://github.com/elvistudio/ai-atlas/issues/15).
- [Relation Target Typing](relation-target-typing.md) — proposes typed relation target objects and target status values. Tracks [issue #16](https://github.com/elvistudio/ai-atlas/issues/16).
- [Level 3 Cross-Branch Placement](level-3-cross-branch-placement.md) — proposes one primary parent, explicit relations, and early standalone private files. Tracks [issue #17](https://github.com/elvistudio/ai-atlas/issues/17).
- [Level 3 Structured Draft Schema](level-3-structured-draft-schema.md) — proposes normalized file-level and node-level fields for private drafts. Tracks [issue #18](https://github.com/elvistudio/ai-atlas/issues/18).
- [Level 3 Capability Categories](level-3-capability-categories.md) — proposes review rules and representation options for capability-oriented candidates. Tracks [issue #19](https://github.com/elvistudio/ai-atlas/issues/19).
- [Process Stage Concept Type](process-stage-concept-type.md) — records Process Stage as experimental and defines current NLG candidate handling. Tracks [issue #20](https://github.com/elvistudio/ai-atlas/issues/20).
- [Level 3 Review and Promotion Process](level-3-review-and-promotion-process.md) — proposes private draft, public draft, and canonical promotion states and gates. Tracks [issue #21](https://github.com/elvistudio/ai-atlas/issues/21).
- [Public L1/L2 Stable ID Storage Design](public-l1-l2-stable-id-storage.md) — compares stable-ID storage options and recommends a review path for public L1/L2 IDs. Tracks [issue #22](https://github.com/elvistudio/ai-atlas/issues/22).

These drafts should be reviewed and revised through their linked issues. Open issues remain the place for unresolved questions and policy decisions.
